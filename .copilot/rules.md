# Правила совместной работы

## Git

- **Коммиты только по явной команде** («делай коммит»). Не коммитить по своей инициативе.
- **Синхронизация `~/friends.test` только по явной команде** («копируй в ~/friends.test»).
- Комментарии к коммитам — на английском, кратко. Длинное описание — только если попросят.
- В каждый коммит добавлять трейлер:
  ```
  Co-authored-by: Copilot <223556219+Copilot@users.noreply.github.com>
  ```
- **Не переписывать историю** (`rebase -i`, `force push`) после начала ревью на Bioconductor.

## Тесты и сборка

- Перед коммитом всегда прогонять `Rscript -e 'testthat::test_local()'`.
- Полная проверка: `R CMD build .` (с виньеткой, без `--no-build-vignettes`),
  затем `R CMD check --as-cran`. Запускать из `R/friends.test/`.
- Целевое состояние: FAIL 0, WARNING 0, не более 2 NOTE (стандартные для Bioconductor).
- BiocCheck: `Rscript -e 'BiocCheck::BiocCheck(".")'` (не `R CMD BiocCheck`).

## Структура репозиториев

- `~/friends-test/R/friends.test` — основная разработка, ветка `devel`.
- `~/friends.test` — копия для Bioconductor, ветка `master` → переименовать в `devel` перед пушем на `git.bioconductor.org`.
- Файлы синхронизируются вручную (`cp`), после чего проверяется `diff -rq`.

## Стиль кода

- Без лишних комментариев — только там, где нужно пояснение.
- Параллельность: `BPPARAM` — opt-in параметр, по умолчанию `SerialParam()`.
- На Unix/macOS — `MulticoreParam`, на всех платформах — `SnowParam`.

## Общение

- Не делать ничего «на всякий случай» без спроса.
- Если что-то неясно — спросить, а не угадывать.
